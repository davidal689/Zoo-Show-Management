Introduction
This document is the authoritative, standalone data dictionary for the Show Management System (SMS), maintained separately from the Software Requirements Specification [SRS] so that data definitions can be found, reviewed, and updated in one place without duplicating or drifting from the shorter summary in SRS §4.2.
Notation
Entries below are either primitive data elements or data structures, listed together in a single alphabetical list. A primitive is a data item that is not broken down further; its Composition or Data Type column states an actual data type, and its Length and Values columns are filled in where applicable. A data structure (record) is composed of other data elements, listed in its Composition column separated by a plus sign (+); the Length and Values columns are left blank for structures. An element shown in parentheses is optional within its structure. An element shown as min:max{element} is a repeating group: it may occur between min and max times, where n means no upper limit. Every element referenced inside a structure's composition has its own entry elsewhere in this alphabetical list, so its definition can always be looked up directly.
